# APA development (operating manual for agents)

Agent-Native Patent Artifact. Read this before editing. The product architecture is in `DESIGN.md`;
the on-disk format is in `docs/protocol.md`; the non-negotiable legal posture is in
`docs/legal-guardrails.md` — **all three are load-bearing.**

## Runtime: Node.js only (no Python, no Bun)

This project targets **Node.js >= 21, plain ESM `.mjs`, zero npm dependencies**. CI runs Node 22. The dev machine has
Node (via nvm4w) but no Python and no Bun, so all MVP tooling must run under plain `node`. (gstack's
Bun-compiled browser/PDF binaries get forked in later phases when those capabilities land; the Phase-1
MVP needs neither.) Use only Node built-ins (`node:fs`, `node:test`, `String.prototype.normalize`, …).

## Commands

```bash
node scripts/gen-skill-docs.mjs            # regenerate canonical Claude skills/*/SKILL.md from .tmpl + resolvers
node scripts/gen-skill-docs.mjs --check    # freshness gate: fail (exit 1) if any Claude SKILL.md is stale
node scripts/gen-skill-docs.mjs --all-hosts  # per-host variants -> dist/<host>/ (claude=skills/, codex/cursor=dist/; gitignored)
node packages/apa-validate/validate.mjs <matter> [--json]   # Level-1 mechanical validation
node packages/apa-viewer/build_manifest.mjs <matter> [--out f]   # build manifest.json for the viewer
node packages/apa-redact/cli.mjs --from-file <f>            # scan-at-sink confidentiality guard
node packages/apa-search/cli.mjs --query "..." --source mock   # prior-art search (offline demo; scans query at sink)
node packages/apa-search/cli.mjs --list-sources            # sources + access modes (api/dataset/ui-restricted)
node packages/apa-assemble/cli.mjs --matter <dir> --write  # assemble filing package + pre-filing gate (no signing/filing)
node packages/apa-rigor/cli.mjs scaffold --matter <dir>    # rigor report skeleton (mechanical dims prefilled)
node packages/apa-rigor/cli.mjs check <report.json>        # validate schema + compute the verdict (deterministic)
node packages/apa-eval/cli.mjs --matter <dir> --mock       # Tier-3 LLM-judge eval (offline; drop --mock + set ANTHROPIC_API_KEY for live)
node packages/apa-prosecute/cli.mjs deadlines --mailed <YYYY-MM-DD>   # post-filing OA response deadlines (estimate)
node scripts/dropin.mjs [--project <dir>]  # zero-install drop-in: wire an APA pointer block into a host project's CLAUDE.md + AGENTS.md (Claude Code / Codex). Idempotent; no skill registration.
node --test "packages/**/*.test.mjs" "lib/**/*.test.mjs" "scripts/**/*.test.mjs" "hosts/**/*.test.mjs"   # full suite (use quoted globs - bare dirs don't recurse in Node 24)
bash build.sh   (or npm run build)         # gen + freshness check + tests + validate example
```

## CI (gstack-style)

`gate.yml` is the blocking, FREE gate (no API key): SKILL.md freshness (`gen-skill-docs --check`),
multi-host generation, the full offline `npm test`, and validation of both worked examples - on Ubuntu
AND Windows. `periodic-evals.yml` runs the PAID `apa-eval` LLM-judge weekly (and on demand), gated on
the `ANTHROPIC_API_KEY` secret so it skips cleanly on forks. The gate uses Node 22+ (the `--test` glob
patterns need >=21). Keep all gate-tier checks offline; anything needing a key or network is periodic.

## SKILL.md workflow (compliance-critical)

`SKILL.md` is **generated** from `SKILL.md.tmpl` + `scripts/resolvers/`. The generated file *is* the
legal-procedure spec, so drift between the rule and the shipped prompt is a liability. Therefore:
1. Edit the `.tmpl` and/or the resolver (never the generated `SKILL.md` directly — it has an
   AUTO-GENERATED header).
2. Run `node scripts/gen-skill-docs.mjs`.
3. Commit both. CI runs `--check` and fails on any drift.

Shared legal rules (preamble, claim-format guide, 101/102/103/112 framing, IDS, drawing standards,
scan-at-sink block) live as **dated resolver tokens** (`scripts/resolvers/`). Update the rule + its
`rulesEffectiveDate` together; the date is surfaced in skill output and artifacts.

## Non-negotiables when editing

- **Mechanical vs. judgment.** The validator may only assert mechanical facts (antecedent basis, claim
  dependency, edge resolution, numeral definedness, type-aware structure). Never make it (or any skill)
  emit a §112-sufficiency or 101/102/103 *conclusion* — those are flags/questions for a human. A green
  check is not a clearance.
- **Unresolved edges are surfaced, never dropped** — in BOTH `build_manifest.mjs` and `viewer.js` (the
  deliberate divergence from ARA's silent edge-drop). A dropped `supported_by` edge is a §112 warning.
- **Structural refusals** (no signing/filing/AI-inventor/micro-entity assertion; no sending unfiled
  disclosure to a non-zero-retention backend without logged consent) are enforced and surfaced in the
  preamble. Don't weaken them.
- **Per-host generation is safety-gated.** A `HostConfig` may rewrite frontmatter and suppress
  *non-safety* resolvers, but the `NON_SUPPRESSIBLE` set (`PATENT_PREAMBLE`, `REDACT_INVOCATION_BLOCK`,
  `ANALYSIS_101_102_103_112`) can never be suppressed — `validateHosts` and `gen-skill-docs` throw if a
  host tries. The post-filing `apa-prosecute` extension still never files and treats deadlines as
  estimates; `apa-eval` never makes a live API call in tests.
- **Legal values are sourced and dated.** Fee amounts, statutory tests, and formatting rules are checked
  against the public-domain authorities under `third_party/uspto-references/` (35 USC, 37 CFR, MPEP, the
  USPTO fee schedule). When you change a legal value (a fee, a rule), cite the source and keep the
  effective-date stamp + "verify currency"; mark anything you couldn't confirm as `_unverified`.
- **No hardcoded paths or fonts; no silent mis-validation.** Unsupported matter types/features fail
  loud ("route to counsel"). Matter-relative paths only.
- **Confidentiality.** An unfiled disclosure is a 102/trade-secret asset. Scan the exact bytes at every
  external sink via `apa-redact`. Real matters live outside `examples/` and are gitignored.

## Matter config rule (platform-agnostic)

Per-matter facts (inventors, `application_type`, jurisdiction, docket, entity status) come from
`PATENT.md` frontmatter. If missing, AskUserQuestion once, then persist — never re-ask.
